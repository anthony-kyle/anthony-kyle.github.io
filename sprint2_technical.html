The Difference Between Relative, Absolute and Fixed Positioning

If you're reading this, at some point you will have come across the CSS property "Position"

position: relative;
position: absolute;
position: fixed;

What is position? and what do these values mean and how do they behave you may be asking? Keep reading to find out

What is Position?

Position is exactly that, the position of a an object on the page. If you think about any object on a website it has a position. Take the following list for example.

  1. This items position is at the top of the list
  2. This item is in second position
  3. This item is at the bottom

You look at a list, especially a numbered list like the one above and you may say, "well obviously the first item is at the top, why do I need a property for that?", and this is an excellent question.

The purpose of the position property, is not to identify where an object is, it is to be able to put the position where you want it to go, and the value you select for it controls how that effects the rest of the page.

A key thing to know when dealing with the position property, is that once you apply a position property to an object it is moved out of the "flow" of the web page. An example of this is if you had a magazine article and you cut a picture out of the page. You could put the image in the exact same place as you cut it out, or you could put it anywhere on the page or even completely off the page. You are able to put the image you have cut anywhere you want without affecting the rest of the page. This is what using the position property achieves on a website.

Relative Positioning

Every object takes up a certain amount of real estate on your screen. Take these photos of a ridiculously cute kitten

They are both positioned normally without having any css positioning applied to them. They both take up the same amount of space, and as html is displaying them natively they simply sit next to one another.

But what if I wanted the photo's to be at different places? I could use relative positioning. Relative positioning shifts an objects location on the page relative to it's starting position. So if I wanted to move the second cat photo down and to the left I would use the following code

/* Set position to be determined relatively */
position: relative;
/* Move the object down 30 pixels from where it started */
top: 30px;
/* Move the object 30 pixels to the right starting from it's left hand edge. */
left: 30px:

Below you can see the effect of this code when applied to just the first cute kitten.

As you can see, the first kitten photo has been moved down 30 pixels and to the right 30 pixels. The thing that you should also notice, is that the padding around the image is now different, with the bottom of the image sticking out through the bottom of the box. This is because as I mentioned above with the magazine example, it is as if you have cut the image out and you are moving it around above the page. So the hole where the image came from is still there, and any padding or margin on the object are still applied to where the hole is, but the image itself is able to be put anywhere with the instructions moving the image from it's original position.

Absolute Positioning

Absolute positioning is very similar to Relative positioning with a simple difference. Positioning an object absolutely completely removes it from it's location on the page and positions it at the specified locations relative to the page.

There is an image in this div.
Don't believe me? View the source.

Did you find the image from the box above? Here's the source code for the box

<div class="example">
 <img class="blog-image absolute" src="images/confused.jpg" alt="This is absolutely positioned.">
 <p>There is an image in this div.<br>Don't believe me? View the source.</p>
</div>

So why can you not see the image? Why is there only the text visible? The answer to both of these is because the image is absolutely positioned. Below I'll show you the css, as well as the image positioned normally.

The CSS

.absolute {
  position: absolute;
  top: 0px;
  right: 0px;
}

The image

Have you seen that image before? Did you follow the code to figure it out? The image is positioned absolutely, on the top right corner of the page. You saw it when you first loaded the page. I told you it would make sense later.

From this example you can see exactly how absolute positioning works. Even though the image was located inside the first div, you could see no trace of it there. Being absolutely positioned removes the entire object from the page, and places it where you specify. The difference between relative positioning and absolute, is unlike relative positioning the other objects in page, move into the place that was previously occupied by the absolute object. Which is why there was no space where the picture came from in the example div.

Fixed Positioning

Fixed positioning works almost identically to absolute positioning. It completely removes the object from flow, and the rest of the page fills in the gap. It then places the object where you tell it. The difference however lies in the fact that a fixed objects position is relative to the screen not the webpage.

"What's the difference between the screen and the webpage?" you may ask, The difference is, the screen is always the screen. The dimensions do not change (unless you resize your window), the top of the screen is always the top, the bottom is always the bottom. No matter how far you scroll on a web page, the bottom of the screen is still the bottom of the screen.

"Okay sure so the screen and the webpage are different. But what does that mean to me?" is yet another excellent question. Remember how in the Absolute example there is the image attached to the top right corner of the page? What happened to that image when you scrolled down? It disappeared, it scrolled off the top of the page as you would expect.

However look at the bottom right of your screen. Do you see a white arrow pointing up on a purple square? What happens when you scroll around the page? Does it disappear? As you have just seen happen, no matter how far you scroll that little purple box stays stuck to the bottom of your screen. It is always there, it is Fixed in place.

The code used on that little tab to get it to stay there is below.

/* Set element to have a fixed position */
position: fixed;
/* Set element to have the bottom attached to the bottom of the screen */
bottom: 0px;
/* Set element to be 5% of the width of the viewport in from the right hand side */
right: 5vw;

I hope that you enjoyed reading this and have formed a stronger understanding on what the css position property does, and how to use it.
